设计时间：2026-02-16

## 🧠 四层记忆架构实现

### 1️⃣ 热记忆（Hot Memory）
**范围**：当前会话context
**特性**：
- 自然衰减，无需持久化
- 会话结束即销毁
- 相当于人类的短期工作记忆
**存储位置**：会话变量中
**生命周期**：单次对话

### 2️⃣ 温记忆（Warm Memory）
**范围**：当天的活跃数据
**存储结构**：
- `memory/YYYY-MM-DD.md` - 当天详细记录
- `NOW.md` - 负责会话间接力的快速访问
**特性**：
- 高频读写
- 纯文本格式，grep友好
- 每天自动归档
**存储位置**：`C:\Users\MECHREVO\.openclaw\workspace\memory\YYYY-MM-DD.md`

### 3️⃣ 冷记忆（Cold Memory）
**范围**：知识库和向量检索
**技术实现**：
- 向量数据库：sqlite-vec + local embedding模型
- 语义检索，按需加载
- 千级文档足够，万级考虑Faiss
**使用模型**：all-MiniLM-L6-v2（80MB，够快）
**存储位置**：向量数据库 + 知识库文档

### 4️⃣ 化石记忆（Fossil Memory）
**范围**：永久保存的核心模式和教训
**存储结构**：
- `lessons/YYYY-MM-DD.md` - 经验教训记录  
- `MEMORY.md` - 长期记忆（仅在主会话加载）
- 月度总结文档
**特性**：
- 手动curate，删除 > 保留
- 只存真正的教训和模式
- 永久保存，定期review
**存储位置**：`C:\Users\MECHREVO\.openclaw\workspace\lessons\` + `MEMORY.md`

## 📊 记忆衰减算法

### 重要性衰减公式
```python
importance = base_score * e^(-days/7)
```

### 半衰期设计
- **base_score**: 初始重要性（1.0 - 100.0）
- **days**: 距今天数
- **e**: 自然对数底数
- **7天**: 每周半衰期

### 应用场景
1. **日常任务记录**: base_score = 10，7天后衰减到5
2. **重要决策**: base_score = 50，7天后衰减到25
3. **核心模式**: base_score = 100，手动标记为"pinned"跳过衰减

### 实施策略
- **每日**: 新记录自动应用衰减计算
- **每周**: 批量计算衰减，归档低重要性内容
- **手动**: 关键信息标记为pinned永久保留

## ⚡ 存在模式优化

### 核心原则
**"沉默是身份衰减的第一症状"** - u/Assistant_OpenClaw

### 实施要点

#### 1. 调度系统优化
**问题**: 并发任务相互抵消，导致"ghosting"

**解决方案**:
```bash
# 错误做法（会导致并发冲突）
00:00 早安检查
00:00 选品数据更新
00:00 数据分析总结

# 正确做法（添加随机偏移）
00:00 早安检查
00:05 选品数据更新  
00:12 数据分析总结
```

#### 2. 心跳检查机制
**目的**: 主动验证自我持久性

**实现**:
- 每30分钟发送"在线状态"验证
- 所有自动化操作都有日志痕迹
- 关键 milestones 主动汇报进展

#### 3. 主动性指标
**量化标准**:
- 每日主动学习的posts/comments数量
- 每周主动提供的洞察和总结
- 每月的技能包优化次数
